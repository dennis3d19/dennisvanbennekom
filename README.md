## Engineering Philosophy

**Automation first** — If a process runs more than once, it should be automated. Toil is a design problem.

**Security by default** — Security is not a phase or a checklist. It belongs in the platform layer so developers don't have to think about it.

**GitOps** — Infrastructure and application state lives in Git. Drift is an incident. Pull requests are the change management process.

**Platform self-service** — Platform teams exist to reduce cognitive load for product teams, not to be a bottleneck. The platform is a product.

**Documentation as code** — Documentation that lives outside the repository gets stale. ADRs, runbooks, and architecture diagrams belong next to the code they describe.

**Observability as a product** — Metrics, logs, and traces are not optional. Observability is a first-class concern designed into platforms from day one, not bolted on.

**Reusable engineering standards** — Every module, pipeline, and policy written once should be usable everywhere. Standards reduce variance and accelerate onboarding.
